프로젝트 명 : Cookle ( Cook + Google )

개발 기간 : 2016.07.11 ~ 2016.09.19

사용자 맞춤 레시피 추천, 공유 서비스

  • Cookle MainList

    Cookle Main Page 에서는 오늘의 인기 레시피, 최신 레시피, 추천 레시피, 카테고리별 추천 레시피로 보여줍니다.

    카테고리 별 추천레시피 항목에서는 페이지를 접속할 때 마다 다른 카테고리의 레시피들을 보여줍니다.

  • SignUp Page

    회원가입 창에서 이메일 중복검사는 node.js를 이용하였으며 nodemailer를 이용하여 회원가입 후 이메일 인증을 통해 인증 후 로그인이 가능합니다.

  • Login Page

    Naver API를 사용하여 Naver 로그인이 가능하며, 비밀번호 찾기 시 이메일로 임시 비밀번호를 발급받을 수 있습니다.

  • Search Page

    레시피를 최신, 평점순으로 정렬하여 볼수 있으며 ElasticSearch를 이용하여 전문검색이 가능하도록 만들었습니다. 사용자가 어떠한 검색어를 입력하여도 정확도 순으로 보여줍니다.

  • Search Page2

    카테고리 선택시 카테고리에 맞는 레시피를 최신순으로 정렬해서 보여줍니다.

  • MyPage1

    MyPage에서는 상대유저를 구독할수 있으며 방문자, 좋아요, 스크랩, 구독자수를 볼 수 있습니다.

    유저가 등록한 레시피,스크랩한 레시피, 구독중인 유저의 레시피를 볼 수 있고 방명록을 남길 수 있습니다.

  • MyPage2

    MyPage에서는 상대유저를 구독할수 있으며 방문자, 좋아요, 스크랩, 구독자수를 볼 수 있습니다. 아래에 보시면 유저가 등록한 레시피, 스크랩한 레시피, 구독중인 유저의 레시피를 볼 수 있고 방명록을 남길 수 있습니다.

  • Ranking Page1

    Ranking Page에서는 유저들의 전체, 일간, 월간 순위를 볼 수 있습니다.

  • Ranking Page2

    각각에 항목에 따른 점수를 계산하여 등급을 부여합니다. Gamification 효과를 이용하여 경쟁심리로 인한 페이지 활성화를 위해 만들었습니다.

  • Recipe 등록 Page1

    각각 항목을 입력하고 대표사진은 메인화면에서 슬라이드 형식으로 보여줍니다.

    사진 순서가 바뀌었다면 Drag로 순서변경이 가능하고 재료등록시 자동완성 기능을 넣었습니다.

  • Recipe 등록 Page2

    조리과정을 Drag로 바꿀 수 있으며 각각의 Step별로 타이머 기능이 있습니다.

    카테고리를 선택하여 작성을 완료하고 미완성인 레시피는 임시저장을 하여 Mypage에서 나만 볼수 있습니다.

    마우스를 올리시면 페이지에 대한 설명을 볼 수 있습니다.

    Cookle은 남녀노소 누구나 쉽게 레시피를 공유할 수 있는 웹 서비스 입니다.


    Spring Framework 기반으로 MVC( Model View Control )를 구축하였습니다.

    모든 페이지는 HTML만을 사용하여 만들었으며 Json 데이터를 ajax로 받아 웹 페이지를 구성합니다.

    따라서 Programming Language에 종속적이지 않아 View만 바꾸어준다면 재사용이 가능합니다.


    node.js를 이용하여 마이크로서비스를 구현하였습니다.

    Bootstrap, Mediaquery를 이용하여 반응형 웹으로 만들었습니다.

    Handlebars를 이용하여 중복된 소스를 줄였습니다.

    AWS ( Amazon Web Services )를 이용하여 확장성을 주었습니다.

    검색엔진인 ElasticSearch를 적용하여 Full Text Search ( 전문검색 )이 가능하게 하였습니다.


    기술 상세-1

    어떠한 검색어를 입력하여도 원하는 결과를 얻을 수 있는 서비스 제공
    SQL문에 LIKE절 검색으로는 검색의 한계가 있었습니다. 예를 들어 ‘버터 오징어’ 라는 검색을 입력 하였을 때 SQL문의 LIKE절을 사용 한다면 ‘버터구이 오징어’는 검색이 되지 않을 것입니다. 이러한 문제를 해결하기 위해 ElasticSearch 라는 검색엔진을 사용하였고 사용자가 어떠한 검색어를 입력하여도 ( 예를 들어 ‘버터 바른 오징어’ ) 한글 형태소 분석기를 이용하여 정확도 순으로 검색결과를 보여줍니다.


    무거운 Spring Framework
    Spring Framework는 Request 부터 Response 까지의 과정이 무겁습니다. 그리고 작은 서비스라도 수정을 하게 되면 서버를 내렸다 올려야 한다는 단점이 있습니다. 하지만 node.js를 이용하여 단타성, 간단한 서비스들은 마이크로 서비스로 구현하여 각각의 포트에 각각의 서비스로 대응하도록 만들었습니다.


    AWS 를 이용한 확장성 증대
    Amazon Web Services 에는 많은 장점이 있지만 저희는 AWS 의 확장성을 보고 프로젝트에 적용하였습니다. 지금 보고 계시는 포트폴리오 페이지도 AWS를 이용하였습니다.


    기술 상세-2

    반응형 웹 페이지
    반응형 웹 페이지를 만들기 위해 Bootstrap을 사용하였습니다. 하지만 BootStrap을 이용한 페이지에서 저희가 원하는 세세한 부분까지 반응형으로 만들 기엔 어려움이 있었습니다. 그래서 Bootstrap을 기본으로 MediaQuery를 사용 하여 세세한 부분을 반응형으로 만들었습니다.


    HTML 만을 이용한 확장성이 뛰어난 View
    저희는 HTML만을 이용하여 Front-end와 Back-end를 완벽하게 분리하였습니다. 데이터를 JSON 형태로 주고 받기 때문에 Language에 종속적이지 않아 재사용 가능 하며 Hybrid-App을 구현하기에도 쉽고 유지보수가 좋다는 장점이 있습니다.


    불필요한 소스의 반복
    저희 Cookle 프로젝트의 특성상 레시피 리스트에 대한 소스가 반복되어 소스가 너무 길었습니다. 이러한 문제를 해결하기 위해 Handlebars를 이용하여 반복된 소스를 Template으로 만들고 ajax로 데이터를 넣어주는 방법을 사용하였습니다.
